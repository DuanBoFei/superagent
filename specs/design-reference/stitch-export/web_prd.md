## 2. 项目背景与目标 (Background & Goals)

### 2.1 背景

SuperAgent CLI 版本已实现核心功能（Agent Runtime / 内置工具 / 权限系统 / MCP / Skills），但终端形态有天然限制：
- 纯文本 UI，代码阅读体验不佳
- 长输出滚动困难
- 缺少图形化的 Diff 对比
- 并行工具进度展示不直观
- 普通开发者的学习门槛较高

现有的 CLI-first 形态限制了 SuperAgent 的触达面，需要一个更友好的 GUI 版本。

### 2.2 目标

打造一个 **CLI 功能的 Web 增强版**，保留所有核心能力，通过 Web UI 大幅提升交互体验。

**一句话定义**：SuperAgent Web 是给**独立开发者**用的**浏览器端编码 Agent 界面**，功能与 CLI 1:1 对齐但体验更好。

### 2.3 非目标 (Non-Goals)

本 MVP **明确不做**：
- ❌ 多用户 / 团队协作
- ❌ SaaS 平台 / 付费系统
- ❌ 云端沙箱执行
- ❌ VS Code / JetBrains IDE 插件
- ❌ 多语言 i18n
- ❌ 桌面 Electron 版本（Web 做完后二期）

## 3. 目标用户与画像 (User Personas)

### 3.1 主画像 P1：独立开发者

| 维度 | 描述 |
|------|------|
| **背景** | 全栈/后端，一个人或小团队，日常写业务代码 |
| **现状工具** | VS Code + GitHub Copilot，用过 Claude Code / ChatGPT 写代码 |
| **核心痛点** | Claude Code 太贵 / 终端改代码看不清 Diff / 长工具输出翻得累 |
| **使用频次** | 每天 10-30 次 Agent 交互 |
| **终端环境** | macOS / Linux，熟悉命令行 |
| **决策权重** | 本地数据安全 > 速度 > 功能丰富度 > UI 美观度 |

### 3.2 次画像 P2：开源维护者

| 维度 | 描述 |
|------|------|
| **背景** | 维护 1-3 个开源项目，issue/PR 积压 |
| **核心痛点** | 想在 iPad / 浏览器里快速处理 issue，不用开终端 |
| **使用频次** | 周末集中使用 |

### 3.3 反画像（明确不服务）

| 画像 | 原因 |
|------|------|
| 非技术用户 / 产品经理 | 这是编码工具，不是通用聊天机器人 |
| 企业级团队协作 | MVP 无团队/权限/审计功能 |
| 纯 Windows 用户 | 优先保障 Unix-like 体验 |

## 4. 用户故事与使用场景 (User Stories)

### US-1: 一键启动 Web 服务

> 作为开发者，当我在项目目录运行 `superagent web` 时，
> 服务自动启动并打开浏览器，我可以直接开始与 Agent 对话，
> 就像在 CLI 里一样。

- **验收**：一条命令启动，零配置，自动打开浏览器
- **边界**：仅限 localhost 访问，不监听公网

---

### US-2: 代码阅读体验升级

> 作为开发者，当 Agent 读取或修改文件时，
> 我可以在聊天流里看到语法高亮的代码块，
> 长代码有滚动条，Diff 用绿/红色清晰展示变更。

- **验收**：代码块有语法高亮、行号、复制按钮；Diff 显示变更前后对比；超过 50 行自动折叠
- **边界**：支持所有主流语言（TS/JS/Python/Go/Rust）

---

### US-3: 多工具并行进度可视化

> 作为开发者，当 Agent 同时调用多个工具时，
> 我可以看到每个工具的实时进度卡片（2x2 网格），
> 完成后自动变成最终结果卡片，不需要翻找。

- **验收**：4 个工具并行时清晰展示各自状态；完成后自动替换为结果；失败标红显示错误
- **边界**：最多同时展示 4 个并行工具，更多的排队并显示"等待中"

---

### US-4: 终端输出彩色展示

> 作为开发者，当 Agent 运行 Bash 命令时，
> 我可以看到 ANSI 彩色的终端输出，
> 长输出有折叠/展开按钮，失败的命令自动展开。

- **验收**：npm test 的彩色输出正确渲染；stderr 标红显示；100 行以上默认折叠
- **边界**：支持 ANSI 256 色，不支持真彩色（够用即可）

---

### US-5: 会话历史与恢复

> 作为开发者，当我刷新浏览器或第二天重新打开时，
> 我可以看到之前的所有会话历史，点击可以继续之前的对话，
> 状态和我上次离开时完全一致。

- **验收**：左侧会话列表按时间倒序；点击加载完整历史；续聊时上下文正确
- **边界**：会话存储在本地 SQLite，不上传到任何服务器

## 6. 详细功能描述 (Detailed Requirements)

---

### F1: Web 服务启动

**触发**：用户在终端运行 `superagent web`

**流程**：
1. 检查 Node.js 环境
2. 启动 Next.js 服务（默认端口 3456，可配置）
3. 自动调用系统浏览器打开 `http://localhost:3456`
4. 终端输出访问地址和日志

**输出**：
- 终端显示服务状态、访问地址、PID
- 浏览器自动加载 Web UI

**边界条件**：
- 端口被占用 → 自动尝试下一个端口（3457, 3458...）
- 浏览器打开失败 → 提示用户手动访问地址
- Ctrl+C → 优雅关闭服务和所有子进程

---

### F2: 聊天流渲染

**触发**：页面加载 / 发送消息 / Agent 流式输出

**核心行为**：
1. 用户消息：右对齐，浅色背景
2. Agent 消息：左对齐，白色背景
3. 流式 token：逐字输出，打字机效果
4. Markdown 渲染：标题、列表、加粗、斜杠、代码块
5. 代码块：语法高亮 + 行号 + 语言标签 + 复制按钮（F10）
6. 长代码：超过 50 行默认折叠，显示"展开全部"按钮（F9）

**边界条件**：
- 消息过长（> 2000 字）：自动截断 + "展开"按钮
- 图片/二进制：不支持，显示"二进制内容无法渲染"
- 渲染失败：降级为纯文本，不崩溃

---

### F3: 工具调用卡片

**通用卡片结构**（所有工具通用）：
```
┌───────────────────────────────────────────┐
│ [工具图标] 工具名 · 状态 · 耗时           │
├───────────────────────────────────────────┤
│                                           │
│   工具结果内容（根据工具类型定制）        │
│                                           │
└───────────────────────────────────────────┘
```

**各工具定制渲染**：

| 工具 | 卡片内容 |
|------|---------|
| **Read** | 文件路径 + 行数 + 语法高亮代码（前 50 行 + 折叠按钮） |
| **Write** | 文件路径 + 写入行数 + Diff 对比（F4 复用） |
| **Edit** | 文件路径 + 变更行数 + Diff 对比（F4 复用） |
| **Bash** | 命令 + 退出码 + 终端彩色输出（F6 复用） |
| **Grep** | 匹配数量 + 文件列表（点击可展开查看匹配行） |
| **Glob** | 匹配文件数量 + 文件树（前 20 个） |
| **Task** | Todo 列表 + 进度条 + 完成/未完成标记 |
| **WebSearch** | 结果列表（标题 + URL + 摘要） |

**状态标识**：
- `⏳ 执行中`：灰色，显示进度条（支持百分比）
- `✅ 完成`：绿色，显示耗时
- `❌ 失败`：红色，显示错误信息
- `⚠️ 被拒绝`：黄色，显示"权限不足，用户已拒绝"

---

### F4: Diff 展示

**触发**：Agent 执行 Write / Edit 工具后

**展示格式**：inline unified diff（不是 side-by-side）
- 新增行：绿色背景 + `+` 前缀
- 删除行：红色背景 + `-` 前缀
- 上下文行：灰色，最多显示前后 3 行

**交互**：
- 鼠标悬停高亮对应行
- 点击"查看完整文件"：展开完整文件对比
- 点击"复制 Diff"：复制 diff 文本到剪贴板

**边界**：
- Diff > 200 行：默认只显示变更片段，完整文件需点击展开
- 二进制文件：显示"二进制文件变更，无法展示 Diff"

---

### F5: 多工具并行进度

**触发**：Agent 一次调用 ≥ 2 个工具

**布局**：2x2 网格（最多 4 个并行）

```
┌─────────────┐ ┌─────────────┐
│ 📄 Read 1   │ │ 📄 Read 2   │
│ ⏳ 60%...   │ │ ✅ 完成      │
└─────────────┘ └─────────────┘
┌─────────────┐ ┌─────────────┐
│ 🔍 Grep     │ │ 📂 Glob     │
│ ⏳ 启动中... │ │ ⏳ 启动中... │
└─────────────┘ └─────────────┘
```

**行为**：
- 每个卡片独立显示进度、状态、耗时
- 完成后自动从"进度卡片"切换到"结果卡片"（F3 的最终样式）
- 超过 4 个工具：前 4 个并行，后面的显示"排队中（#5/#6/#7...）"

---

### F6: 终端彩色输出

**触发**：Agent 执行 Bash 工具

**渲染**：
- ANSI escape code 转 HTML 样式（前景色 / 背景色 / 粗体 / 下划线）
- 等宽字体，和终端对齐
- 保留换行和空格格式

**折叠逻辑**：
- ≤ 50 行：默认展开
- 51-200 行：默认展开，显示"折叠"按钮
- > 200 行：默认折叠，显示"展开全部（xxx 行）"按钮
- 命令失败（非 0 退出码）：强制展开，标红背景

---

### F7: 会话历史列表

**位置**：左侧边栏（固定宽度 240px）

**内容**：
- 会话标题：自动提取第一条用户消息的前 20 字
- 时间：相对时间（"5 分钟前" / "昨天" / "2026-06-15"）
- 状态图标：进行中 / 已完成 / 失败
- 最新消息预览：最后一条消息的前 30 字

**交互**：
- 点击加载对应会话
- 右键菜单：重命名 / 删除
- 顶部搜索框：按关键词搜索会话历史

---

### F8: 流式消息输入

**位置**：底部固定输入框

**功能**：
- 支持多行输入（Shift+Enter 换行）
- 支持粘贴文本/代码
- 发送按钮（Enter 发送，Shift+Enter 换行）
- 输入历史：上下键切换历史输入
- 自动增高：最多增高到 30% 屏幕高度，超过出现滚动条

**快捷按钮**（输入框左侧）：
- 📎 附件：上传文件（写入临时目录，供 Agent 读取）
- 📁 选择文件：从本地项目目录选择文件
- 🔄 清空输入

## 7. 非功能需求 (Non-Functional Requirements)

### 7.1 性能

| 指标 | 目标值 |
|------|--------|
| 服务启动时间 | ≤ 2 秒（`superagent web` 到浏览器打开） |
| 首屏渲染 | ≤ 1 秒 |
| 消息流式渲染延迟 | ≤ 100ms（token 到达 → 显示到屏幕） |
| 工具卡片渲染 | ≤ 50ms |
| 会话历史加载 | ≤ 500ms（100 个会话） |
| 热键响应 | ≤ 50ms（按下 → 响应） |

### 7.2 安全

| 要求 | 说明 |
|------|------|
| 仅限 localhost | 默认只监听 127.0.0.1，不监听 0.0.0.0 |
| 无公开端口 | 禁止公网访问（后续可加 Token 认证解锁） |
| API Key 安全 | Model Key 只存在本地磁盘，永不发送到第三方（除了模型服务商） |
| XSS 防护 | 用户输入/工具输出严格转义，React default |
| CSRF 防护 | Socket.io 内置机制 |

### 7.3 可靠性

| 要求 | 说明 |
|------|------|
| 前端崩溃恢复 | React Error Boundary 捕获异常，显示友好错误页，自动重试 |
| 后端崩溃自动重启 | 进程守护，崩溃后 1 秒内自动重启（保留会话） |
| 网络断开自动重连 | WebSocket 断开后每 3 秒重试，重连成功恢复会话 |
| 消息至少一次送达 | 消息 ID + 确认机制，不丢消息 |

### 7.4 兼容性

| 浏览器 | 最低版本 |
|--------|---------|
| Chrome | ≥ 100 |
| Firefox | ≥ 100 |
| Safari | ≥ 16 |
| Edge | ≥ 100 |

**不支持**：IE、移动端浏览器

## 8. UI / 交互说明 (UI / UX)

### 8.1 整体布局

```
┌─ 240px ─┬───────────────────────────────────────────┐
│          │                                           │
│  左侧边栏 │        主聊天区域（70% 屏幕高度）        │
│  - 会话列表 │                                           │
│  - 新项目  │                                           │
│          │                                           │
│          ├───────────────────────────────────────────┤
│          │        输入区域（固定底部 30%）           │
│          │  [📎] [多行输入框]            [发送]      │
└──────────┴───────────────────────────────────────────┘
```

### 8.2 设计风格

- **主题**：深色优先，浅色可选（F12）
- **配色**：低饱和专业风格，参考 VS Code Dark+
- **字体**：UI 用 Inter，代码用 JetBrains Mono
- **圆角**：8px（卡片）/ 4px（按钮）
- **阴影**：软阴影，不要夸张效果
- **动效**：≤ 200ms 过渡动画，避免花哨

### 8.3 关键交互原则

1. **零模态**：所有操作都在当前流完成，不弹窗打断用户
2. **渐进式披露**：默认显示最重要的信息，详情需要时再展开
3. **状态可追溯**：每个工具调用都要显示"成功/失败 + 耗时"
4. **无惊喜**：操作结果可预期，不要出现用户没点过的自动跳转

## 9. 验收标准 (Acceptance Criteria)

### F1: Web 服务启动

**Given** 用户已安装 superagent CLI
**When** 用户在项目目录运行 `superagent web`
**Then**
- 服务在 2 秒内启动
- 自动打开默认浏览器访问 `http://localhost:3456`
- 终端显示访问地址和服务日志

**Given** 3456 端口已被占用
**When** 用户运行 `superagent web`
**Then** 自动尝试 3457，直到找到可用端口

---

### F2 & F3: 聊天流 + 工具卡片

**Given** 用户在聊天框输入问题
**When** 用户点击发送
**Then**
- 用户消息立即出现在聊天流右侧
- Agent 开始流式输出响应
- 遇到工具调用时，渲染对应类型的工具卡片
- 工具卡片显示状态（执行中/完成/失败）和耗时
- 工具完成后显示完整结果

---

### F4: Diff 展示

**Given** Agent 建议修改 src/runtime.ts 第 40-45 行
**When** 工具执行完成
**Then**
- 聊天流中出现 Diff 卡片
- 新增行是绿色 + `+` 前缀
- 删除行是红色 + `-` 前缀
- 点击"复制 Diff"按钮，剪贴板内容正确

---

### F5: 多工具并行

**Given** Agent 一次调用 4 个 Read 工具
**When** 工具并行执行
**Then**
- 聊天流中出现 2x2 网格的进度卡片
- 每个卡片独立显示进度百分比
- 完成后自动替换为结果卡片
- 4 个工具总并发执行时间 < 最慢单个工具 × 2

---

### F6: 终端输出

**Given** Agent 运行 `npm test` 产生 150 行彩色输出
**When** 命令执行完成
**Then**
- 卡片中显示 ANSI 彩色输出
- 默认折叠，显示"展开全部（150 行）"
- 点击展开，输出完整显示
- 如果测试失败（非 0 退出码），自动展开并标红

---

### F7: 会话历史

**Given** 用户之前有 10 个历史会话
**When** 刷新页面
**Then**
- 左侧边栏显示 10 个历史会话（按时间倒序）
- 点击任意会话，聊天区域正确加载当时的所有消息
