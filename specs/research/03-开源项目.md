| **插件系统** | 5/5 | `src/plugins/` + `src/plugin-sdk/` -- 三层扩展（Hooks/Tools/Channels/Providers/HTTP/GatewayHandlers）；`extensions/` 扩展工作区；运行时可加载/卸载插件 |
| **多渠道网关** | 5/5 | Gateway 统一接入 Telegram/Slack/Discord/Signal/iMessage/WhatsApp/Web 等；路由层 (channel/account/peer -> agent/session)；SessionKey 跨模块隔离主键 |

**最值得参考的设计模式**：
1. **Gateway 四层架构**: 接入层 → 编排层 → Agent Runtime 核心层 → 能力子系统层 -- 清晰的关注点分离
2. **双队列并发模型**: Session lane + Global lane -- 同一会话串行保证上下文一致性，全局可控并发
3. **插件扩展体系**: Plugin SDK + 运行时注册表 + 扩展工作区 -- 可扩展性最佳实践
4. **SessionKey 隔离机制**: 跨模块 (路由/并发/会话存储/记忆召回/事件分发) 统一使用 SessionKey 作为隔离主键
5. **Tool 三层架构**: Coding tools / Agent tools / Plugin tools -- 不同来源的工具分级治理

## 6. 关键发现与建议

### 6.1 核心发现

1. **MCP 已经成为行业标准协议**。2026 年几乎所有主流 Agent 框架都原生支持 MCP，Google 的 A2A (Agent-to-Agent) 作为补充正在兴起。MCP 用于 Agent-to-Tool 通信，A2A 用于 Agent-to-Agent 通信。我们的项目应将 MCP 作为一等公民支持。[WebSearch] [双路确认]

2. **Python 是 Agent 框架的主流语言**。虽然 Go (OpenCode) 和 Rust (Codex CLI, Goose) 在性能上有优势，但 Python 在 AI/ML 生态、开发效率、社区支持方面占据绝对主导。Hermes Agent 用 Python 构建了最完整的 Agent 功能集，直接验证了 Python 用于生产级 Agent 的可行性。[本地]

3. **"写代码式" Agent 正在超越 JSON Tool Call**。smolagents 的 CodeAgent (LLM 直接写 Python 代码执行而非每步 JSON round-trip) 将 5 步 workflow 从 5 次 API 调用压缩到 1 次。[WebSearch] 这是一个值得关注的设计范式。

4. **上下文管理是 Agent 的核心竞争壁垒**。Letta (OS 启发式记忆) 和 Hermes (四层分层记忆) 代表了两种主流路线：Agent 自主管理记忆 vs 结构化分层记忆。Hermes 的四层记忆工程化最完整。[本地] Mastra 的"观测压缩式"在 LongMemEval 上达到 SOTA 94.87%，证明了"暴力压缩到上下文窗口"在 2026 年的可行性。[WebSearch]

5. **Claude Code 的 Tool/Task/Agent 三层抽象是 Harness 设计的最佳参考**。`QueryEngine.ts` 的消息循环 + `Tool.ts` 的接口规范 + `Task.ts` 的生命周期管理 -- 这三个文件包含了一个生产级 Agent Runtime 90% 的核心设计.[本地]

6. **OpenTelemetry 是 2026 年 Agent 可观测性的事实标准**。Agent-native span semantics (agent_step, tool_call, llm_call, guard) 正在形成共识。Agent Observatory, Plyra Trace, AgentWeave 等新项目在快速建立 "Agent OTel" 的语义规范。[WebSearch]

7. **Rich + prompt_toolkit (或 xli) 是 Python Agent CLI 的最佳组合**。Hermes Agent 已用 rich==14.3.3 + prompt_toolkit==3.0.52 证明了此方案的可行性。[本地] xli 提供了更高层的 Agent REPL 封装，可将 TUI 开发从 ~2000 行缩减到 ~20 行。[WebSearch]

8. **生态"三国杀"格局**：OpenCode (开源编码Agent, 170k stars) / Claude Code (非开源但源码可用, 124k stars) / Codex CLI (Rust 高性能, 90k stars) 主导终端编码Agent赛道。通用Agent框架由 LangGraph/CrewAI/AutoGen 三强分割。[WebSearch]

### 6.2 对"从 0 自己写"的建议

1. **语言选 Python**。Python 3.11+ 生态最丰富，Hermes Agent 已充分验证 Python 用于生产级 Agent 的可行性。[本地]

2. **Agent Core Loop 设计参考 Claude Code 的 QueryEngine + Tool + Task 三层抽象**，但用 Python 实现。[本地]

3. **上下文管理直接参考 Hermes Agent 的 context_compressor.py + conversation_compression.py + memory_manager.py**。[本地] 再结合 Letta 的 OS 启发式记忆理念做创新。[WebSearch]

4. **MCP 用官方 Python SDK (`mcp[cli]`)** 做 MCP Client，参考 IBM chuk-mcp 做高性能 MCP Server，参考 smolagents 的 `ToolCollection.from_mcp()` 做集成范式。[WebSearch]

5. **TUI 用 Rich + prompt_toolkit + xli**，已被 Hermes Agent 验证的生产级方案。[本地] [WebSearch]

6. **可观测性一开始就内建 OTel**，使用 OTel Python SDK + Agent-native span semantics (agent_step/tool_call/llm_call)。不要等到后期再补。[WebSearch]

7. **不要从零实现 Browser Use**，通过 MCP 协议接入 browser-use 即可获得浏览器能力。[WebSearch]

8. **插件/Skill 系统参考 OpenClaw 的 Plugin SDK + Hermes Agent 的自我进化 Skill**。[本地] 这是两个不同哲学（静态插件 vs 动态进化），可结合两者优点。

9. **Task 规划参考 LangGraph 的 StateGraph 概念但不直接依赖**。实现轻量级 Checkpointer + StateGraph 即可满足大部分需求，无需引入整个 LangChain 生态。[WebSearch]

10. **沙箱执行参考 Codex CLI 的 OS 级 sandbox 设计**。[WebSearch] 初期可用 Docker 容器，后期加 OS 级沙箱。
