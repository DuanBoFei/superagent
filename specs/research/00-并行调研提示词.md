## 调研主题 2：API 与模型服务

本 Agent 的关键资源是**模型 API**——需要支持多模型、多 Provider。

### 2.1 模型 Provider 盘点

至少 8 家可对比：

| Provider | 需调查 |
|----------|--------|
| Anthropic | Claude Opus 4.5 / Sonnet 4.5 / Haiku 4.5，Prompt Cache 机制与定价 |
| OpenAI | GPT-4o / GPT-4.1 / o4-mini / Codex，Chat Completions vs Responses API |
| Google | Gemini 2.5 Pro / Flash，免费额度，Vertex AI vs Google AI |
| DeepSeek | DeepSeek-V3 / R1，国内访问、价格优势 |
| 通义千问 | Qwen 系列，国内合规、价格 |
| 智谱 | GLM 系列，国内合规 |
| Moonshot | Kimi，长上下文特色 |
| xAI | Grok，最新模型 |

### 2.2 每家 Provider 的关键事实

- 价格档位（input/output token 价格、免费额度）
- Prompt Cache 支持（是否支持、命中率、价格折扣）
- SLA / 可用性 / 限流（QPS、TPM 限制）
- 区域可用性（国内直连 vs 需要代理）
- 合规（数据驻留、中国大陆可用性）
- API 兼容性（是否兼容 OpenAI SDK 格式？切换成本）
- 模型特性对比（上下文窗口、Function Calling 质量、代码能力）

### 2.3 个人开发者主流选择

- 中文开发者社区最常用哪几家？组合策略是什么？
- 贵模型（Opus/GPT-4o）做什么、便宜模型（Haiku/Gemini Flash）做什么？
- 有没有成熟的 API 网关/路由方案（如 LiteLLM、one-api）？

### 2.4 推荐方案

- 主方案（主力模型）+ 备份方案（failover）
- 成本优化策略（Prompt Cache + 分层模型调度）
- 国内网络环境下的最佳实践

产出：`02-API与模型服务.md` —— Provider 对照大表 + 推荐组合 + 风险点
